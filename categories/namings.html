Naming Conventions

If you are wondering what do things like g_fl, Ws or Ts mean: usually this is how some devs indicate which type is expected, like for variables or functions. This isn't necessary by any means, you can give your functions and variables any name. (although having consistent naming convention is nice if you're working in a team).

List of Common Definitions

  • g_ prefix usually stands for "global". Usually this is how devs indicate that data is accessible from anywhere, like global variables.
  • _fl prefix stands for float. It indicates that this is a single-component value specifically, not a vector float.
  • _i prefix stands for int.
  • _v usually just stands for a vector, which can be float2, float3 or float4. Maybe int vector too.
  • _mat indicates that this is a matrix variable.
  • Os suffix stands for object space. Before vertices are transformed into world-space/projection space coordinates, usually they are sent to shaders in object space coords first.
  • Ws suffix stands for world space. Like world-space normals, world space pixel position, anything that describes any data in world coordinates.
  • Ps suffix is projection space. You could see this in vertex struct. Before finishing vertex shader, you're expected to transform your vertices into projection space. So functions called something like Position3WsToPs( ... ); means "convert float3 world-space position into projection space".
  • Ss suffix is screen space. PixelInput struct contains i.vPositionSs, which is essentially UV for your current camera's viewport. Even if you're making a surface shader for your mesh, this viewport UV, unlike the rest of your mesh, is actually in screen-space coordinates.
  • Ts suffix is tangent space. Usually this is how you may want to name your tangent-space normal maps variables, that are not transformed into world space normals yet.

Automatic Variable Cleanup

Material Editor comes with a little handy feature: if you have variables that follow this naming convention, and they are also exposed for editing by artists, any specific prefixes like g_fl will be automatically removed from the name of this variable in material editor UI.

It will also automatically format variable name into a readable format. For example, g_vMyVectorWs will be displayed in Material Editor UI as "My Vector".
