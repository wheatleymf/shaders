This is a list of changes on this website. If there's any significant update to the contents, they will be probably shown here. Additionally, you can monitor commit changes on GitHub to see all exact changes. This page serves it's purpose as a summary for most important commits.

2025

September 2025

September 4th, 2025:

  • Updated article header styling. Should stand out a bit more now. :-)
  • Category navigation panel has been slightly re-aligned. If you stay at the top of the page, it will be slightly offset to avoid overlapping with the header section. If you scroll below, it will go back to it's original position.
  • Navigation bar on the left is now hidden when your screen width is less than 800px, currently there's no way to make it visible if your screen is too small, but I can probably change this behavior if requested.

August 2025

August 28th, 2025:

  • Removed CalculateInstancingObjectToWorldMatrix from function page, since this function no longer works.
  • Added references to GetTransformMatrix and ComputeMatrixScale on common functions page which allow extracting object's transform matrix, and quickly calculating model's scale.
  • Removed "Vertex Stuff" category from common functions page, and moved Position3WsToPs to Miscellaneous. Updated the description to be more correct, since this function sometimes can be used outside of vertex shaders.
  • Updated Relics of the Past page with new information about CalculateInstancingObjectToWorldMatrix. Described what happened to this function, and added a code snippet that shows how modern shaders can get model's transform matrix now.

July 2025

July 31st, 2025:

  • Functions page has been updated to provide slightly more detailed description on various BRDF-related functions and arguments that they take.

    I've also added reference to AnisoGGX, which is used to calculate anisotropic specular lobe. Added reference to a function which allows getting object-to-world matrix for current instance.

June 2025

  • Updated HLSL highlighter to support some s&box specific names.
  • Updated CSS to support infinitely tall navigation column.
  • Added new article: Skybox Shader

I don't have much time to update the article currently, but this will change soon, hopefully! I apologize for the inconvenience.

January 2025

January 31st, 2025:

  • Implemented loading screen boilerplate. It will hide all ugly missing icons and font before page was fully loaded. If user has encountered an error during load that gets page stuck in loading state forever, they will be notified in 20 seconds. This screen is visible only during the initial page load.
  • Navigation menu will no longer show up when user is sliding around the code snippets in articles. Improved code organization for handling navigation in mobile mode.
  • Changed the way h2/h3 tags are wrapped to prevent them rendering beyond phone's viewport maximum width. This isn't perfect but should fix text rendering across all potential browsers.

January 30th, 2025:

  • Implemented basic mobile phone support. This should greatly improve the quality of life for mobile users, at least layout should not be fundamentally fucked anymore. Swipe to the right to open a navigation menu, swipe to the left to hide it. This is an early iteration so please let me know if anything breaks.

    Night theme switch button will be displayed only when you have navigation bar open. At the moment, quick navigation with headings on the right isn't available on mobile phones.

    Current method for detecting mobile phones is done by checking if user has appropriate viewport width, if you're using tablets/desktops and see mobile layout enabled anyway, please let me know.

  • Updated all shader files to include Depth mode. This was an oversight on my end, this must be featured by default because shadows won't cast without depth prepass. Sorry!
  • Updated Your First Unlit Shader. s&box will change the name of the main rendering mode from VrForward to Forward. Added a reference to "Relics of the Past" article. Fixed typo. Added a very little notation about forward+ rendering.
  • Shader files have also been updated to use Forward mode from now on.
  • Relics of the Past article has been updated to feature old mode VrForward and what should be used instead of it from now on.
  • Added a dedicated changelog page that you're currently reading. :-)
