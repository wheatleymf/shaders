Before you start

This is the beginning of a Shader Fundamentals course. Before we start, it's important to note some stuff first. This course will be painfully obvious, and some things will be explained with unholy amount of detail - I am expecting that you have nearly zero understanding of shaders. This is a very early iteration of this course, and some things may be incorrect, or poorly written - please be patient as I try to polish this thing.

References

If you plan to continue learning shaders on your own, you must know where you can find references for s&box shaders. There's no Intellisense so I highly recommend looking up existing code.

  • s&box shaders location: Steam/steamapps/common/sbox/addons/base/assets/shaders
  • s&box Documentation on shaders. This documentation covers lots of s&box shader API, helper functions and other useful things.
  • s&box wiki. It isn't always up-to-date, but it never hurts to check it out.
You probably already know about websites like The Book of Shaders or Shadertoy so I won't suggest them here. Microsoft also provides a complete documentation for nearly everything what HLSL has. This probably won't be very useful for you as a beginner, but once you figure out the basics I recommend visiting it for any specific things.

Default lighting model that s&box uses relies on PBR, aka Physically Based Rendering. This means that you should probably be familiar with default artist's PBR workflow, understanding why do you need such texture maps as albedo, roughness, normals, metalness, etc... If you'd like to get extremely nerdy about it, "PBR Book" should be very helpful.

And another tip: take your time. If something isn't clicking in your head and you can't figure out some concept, try again later. Learning new things can fuck with your sanity, and sometimes the best solution is just coming back to it later.

Do I need to understand programming?

Yeah. HLSL isn't a good language to start your programming adventures. If you have some experience with C#, you should feel comfortable with 99% of HLSL syntax. Just like most other languages, is is made of variables, classes, functions and statements. You can define custom functions and classes, do any sorts of maths using same operators

Software Recommendations?

It doesn't really matter, so pick any text editor you prefer. Some people use Visual Studio, some prefer to work in Visual Studio Code. Stuff like JetBrains Rider probably also works. If you're using VS Code, there are bunch of extensions that can improve HLSL code markdown to make navigating through code much easier.

IntelliSense

As of now, there's no IntelliSense for HLSL code or s&box API. This may appear annoying at first, but there's a good thing about s&box API - it doesn't rely on internal functions and API too much. You won't need to remember hundreds of random macros, functions, or anything else. With enough practice, you'll feel comfortable typing code without having to constantly look up docs or some reference shaders.
